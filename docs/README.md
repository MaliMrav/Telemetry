# Documentation

This directory contains the architectural, historical, reference, and roadmap documentation for Telemetry.

## Start Here

- [The Why](The-Why.md) — Why Telemetry exists.
- [How We Think](How-We-Think.md) — The principles guiding architectural decisions.
- [Architecture](ARCHITECTURE.md) — The current system architecture.

## Architecture Journey

The `architecture/` directory records the major architectural pressures and the decisions made in response to them.

## Reference

The `reference/` directory contains stable technical explanations of specific architectural areas.

## Roadmap

The `roadmap/` directory records the project's architectural journey through sprints.

The sprint documents explain the objective, architectural pressure, decisions, principles exercised, and breadcrumb to the next decision.

The goal is to help a reader reach the moment:

> "Ahh... now I understand why."
